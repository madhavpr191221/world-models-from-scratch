# Project Constitution

This directory defines the project-level contract for SpecDD.

The constitution is intentionally stable. Feature work should inherit from it rather than re-litigate these decisions every session.

## Files

- [Mission](./mission.md)
- [Tech Stack](./tech-stack.md)
- [Roadmap](./roadmap.md)

## Workflow

1. Read the constitution first.
2. Pick the next roadmap item.
3. Write a feature spec under `docs/specs/<area>/`.
4. Implement on a feature branch.
5. Validate with the required scripts, plots, and reports.
6. Merge only after the spec and implementation agree.

## Core Principle

The spec explains the what and why.
The code implements the how.
